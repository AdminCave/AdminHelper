Go-Agent

Unified Go-Agent unter agent-go/. Ersetzt die alten Shell- und Python-Agents und vereint Monitoring, FRP-Client-Setup und Service-Installer in einer einzigen Binary, die gleichermaßen unter Linux (systemd) und Windows (Service-Manager) läuft.

Projekt-Layout

tree
agent-go/
├── cmd/adminhelper-agent/   main.go – Cobra-Root
├── internal/
│   ├── cli/                 Subcommands (run, provision, monitor, ...)
│   ├── config/              agent.toml + Env-Var-Overrides
│   ├── http/                API-Client mit Auto-Retry
│   ├── collectors/          cpu, ram, disk, smart, docker, zfs, proxmox
│   ├── frp/                 frpc-Verwaltung, Cert-Rotation
│   └── service/             systemd / windowssvc-Integration
├── systemd/                 .service + .timer Units
├── rpm/                     Spec-File
├── build-deb.sh
├── build-rpm.sh
└── go.mod

Cobra-Subcommands

CommandZweck
versionVersion + Git-Commit ausgeben (gesetzt über -ldflags).
provisionErst-Setup: Server-URL, Token einlösen, Zertifikat holen, Services aktivieren.
runVollständigen Agent-Zyklus einmal durchspielen (Monitoring + FRP).
monitor runNur Monitoring-Collector-Lauf.
frpc rotateClient-Zertifikat manuell erneuern.
service install / uninstallsystemd-Unit- bzw. Windows-Service-Installation.

Collector-Framework

Jeder Collector implementiert ein simples Interface:

go
type Collector interface {
    Name() string
    Enabled(cfg *config.Config) bool
    Collect(ctx context.Context) ([]Sample, error)
}

type Sample struct {
    Metric string
    Value  float64
    Labels map[string]string
    Time   time.Time
}

Vorhandene Collectors:

  • cpu – Load + Per-Core-Utilization (aus /proc/stat bzw. GetSystemInfo auf Windows)
  • ram – Total/Used/Free/Swap
  • disk – pro Mountpoint
  • smart – via smartctl --scan-open + --all --json=cv
  • docker – via Docker-API-Socket
  • zfs – via zpool status -p
  • proxmox – via pveversion + API (nur auf PVE-Hosts)

systemd-Timer mit Jitter

Statt cron nutzt der Agent systemd mit OnCalendar=*:0/5 + RandomizedDelaySec=30s. Der Jitter verteilt die Last aller Agents gleichmäßig auf das 5-Minuten-Fenster und verhindert Thundering-Herd-Requests am Server.

Build

bash
cd agent-go
GOOS=linux   GOARCH=amd64 go build -o bin/adminhelper-agent     ./cmd/adminhelper-agent
GOOS=windows GOARCH=amd64 go build -o bin/adminhelper-agent.exe ./cmd/adminhelper-agent

VERSION=0.18.0 bash build-deb.sh
VERSION=0.18.0 bash build-rpm.sh
DEB/RPM Replaces

Die DEB/RPM-Pakete enthalten Replaces- und Conflicts-Einträge für die Legacy-Pakete srm-frpc-client, srm-monitor-agent und srm-agent. Ein apt install adminhelper-agent migriert saubere Installationen automatisch.
