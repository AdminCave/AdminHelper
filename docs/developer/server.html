Server (Backend)

Der Server ist ein FastAPI-Monolith: HTTP-Router, SQLAlchemy-Models, APScheduler-Jobs und FRP-PKI laufen im selben Prozess. Separation by Module, nicht by Deployment.

Einstiegspunkte

DateiZweck
server/app/main.pyFastAPI-App, Middleware-Stack, Static-Mounts für Vite-Output.
server/app/core/config.pyEnv-Var-Resolution, JWT-Settings, Secret-Key-Persistence.
server/app/core/auth.pyJWT-Encode/Decode, Refresh-Rotation, Blacklist.
server/app/core/db.pySQLAlchemy-Engine + Session-Factory.

Auth-Architektur

JWT-Access-Token (HS256, 8h) im Authorization-Header; Refresh-Token (7d) als httpOnly-Cookie. Bei jedem Refresh:

  1. Server prüft Signature + Ablauf
  2. Alter jti wird in die Blacklist-Tabelle geschrieben
  3. Neuer Access + neuer Refresh werden ausgestellt
  4. Wird ein geblacklisteter jti erneut verwendet → Reuse-Detection: alle Sessions des Users werden invalidiert.
Rate-Limit

Login-Rate-Limit ist Redis-basiert: Key ratelimit:login:<ip>. Fallback auf In-Memory-Dict (nur bei Single-Worker sicher). Implementierung in server/app/core/rate_limit.py.

SQLAlchemy-Models

Die wichtigsten Tabellen leben in server/app/db/models/:

  • User, Session, RefreshBlacklist
  • Connection, ConnectionGroup
  • Server, ProvisionToken, FrpClientCert
  • ApiKey (inkl. scope JSON-Array)
  • MonitoringCheck, MonitoringResult, Alert
  • Hook, HookExecution
  • AuditEntry

Scheduler

APScheduler läuft im selben Event-Loop wie FastAPI. Job-Store in der DB (SQLAlchemy-JobStore), Trigger-Typen: Interval (Monitoring-Checks), Date (einmalige Hook-Retries), Cron (optional für Reports).

Hook-System

Hooks reagieren auf Events wie server.provisioned oder alert.triggered. Ausführung erfolgt in einem Subprozess mit begrenzter Laufzeit und sanitisierter Umgebung – Details siehe Hooks & Events.

Tests

Pytest-Suite unter server/tests/. Test-DB wird per DATA_DIR=/tmp/... in conftest.py isoliert. FastAPI-TestClient nutzt einen shared-memory SQLite, damit Migrationen deterministisch sind.

bash
cd server
uv run pytest -x                 # schnell abbrechen bei Fehler
uv run pytest --cov=app          # Coverage
