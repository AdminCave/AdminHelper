AdminHelper Agent — Reference

The AdminHelper Agent is a unified Go binary for FRP tunnel synchronization and system monitoring on Linux and Windows.

Installation Command Reference

1) Overview

The adminhelper-agent is a unified Go binary that fully replaces the legacy shell and Python scripts. It combines two main functions in a single program:

FRP Client Synchronization

Automatic synchronization of the FRP client configuration with the AdminHelper server. The agent downloads tunnel definitions, PKI certificates and automatically generates the frpc.toml.

System Monitoring Push

Collects system metrics (CPU, RAM, disk, services, temperatures) and sends them via HTTPS push to the monitoring service. No inbound ports required.

The agent runs as a systemd timer (Linux) or Windows service and executes a synchronization and monitoring cycle every 5 minutes. Supported platforms: Linux amd64 and Windows amd64.

2) Installation

DEB Package (Debian/Ubuntu)

sudo apt install ./adminhelper-agent_0.13.0_amd64.deb

Installs the agent to /usr/local/bin/ and creates systemd unit files.

RPM Package (RHEL/AlmaLinux)

sudo rpm -i adminhelper-agent-0.13.0-1.x86_64.rpm

Analogous to the DEB package — agent binary and systemd units are installed.

Windows Binary

Copy the EXE file to C:\Program Files\AdminHelper\ and register it as a Windows service: 

copy adminhelper-agent.exe "C:\Program Files\AdminHelper\"
"C:\Program Files\AdminHelper\adminhelper-agent.exe" service install

3) Command Reference

Command Description
adminhelper-agent version Display version
adminhelper-agent run Daemon mode: FRP sync + monitoring every 5 minutes
adminhelper-agent run --once Single execution (for systemd timer)
adminhelper-agent frpc init Initial FRP client setup. Flags: --url URL --token TOKEN --server-id ID [--cacert PATH] [--insecure]
adminhelper-agent frpc sync One-time configuration synchronization
adminhelper-agent monitor init Initial monitoring setup. Flags: --url URL --api-key KEY --server-id ID [--services svc1,svc2] [--cacert PATH] [--insecure]
adminhelper-agent monitor push One-time metric push
adminhelper-agent service install Register systemd timer or Windows service
adminhelper-agent service uninstall Unregister service

4) Configuration Files

Both configuration files use a simple KEY=VALUE format (one variable per line).

FRP Configuration (adminhelper.conf)

  • Linux: /etc/frp/adminhelper.conf
  • Windows: %ProgramData%\AdminHelper\frp\adminhelper.conf
# /etc/frp/adminhelper.conf
ADMINHELPER_URL=https://admin.example.com
API_KEY=abcdef1234567890abcdef1234567890
SERVER_ID=srv-web-prod-01
CACERT=/etc/frp/ca.crt
INSECURE=false

Monitor Configuration (monitor.conf)

  • Linux: /etc/adminhelper/monitor.conf
  • Windows: %ProgramData%\AdminHelper\monitor.conf
# /etc/adminhelper/monitor.conf
MONITOR_URL=https://admin.example.com/api/monitoring
API_KEY=abcdef1234567890abcdef1234567890
SERVER_ID=srv-web-prod-01
SERVICES=nginx,postgresql,redis
CACERT=/etc/adminhelper/ca.crt
INSECURE=false
Note: The configuration files are automatically created by frpc init and monitor init respectively. Manual editing is only necessary for special cases.

5) Provisioning Flow

  1. Create the server in the AdminHelper Web UI.
  2. Generate a provision token (Web UI or API).
  3. Initialize the agent on the target server: 
    sudo adminhelper-agent frpc init \
  --url https://server:443 \
  --token TOKEN \
  --server-id SERVER_ID
  4. The agent downloads the PKI bundle (CA certificate, client certificate, private key).
  5. The agent creates adminhelper.conf and frpc.toml.
  6. Start the timer/service: 
    sudo adminhelper-agent service install
  7. Automatic synchronization runs every 5 minutes from this point on.
Tip: Monitoring can be set up in the same step — simply run monitor init in addition, before calling service install.

6) Monitoring Collector

The integrated collector captures the following system metrics on each push cycle:

Metric Details
CPU Utilization in percent (1-second average)
Memory Usage in percent, total MB, used MB
Load 1m, 5m, 15m average (Linux only)
Disks Per partition: mount point, filesystem, usage %, total/used GB. Automatically filters: squashfs, tmpfs, devtmpfs, overlay, proc, sysfs
Uptime System uptime in seconds
Services Status of configured services (systemd on Linux, Windows Services)
Temperatures CPU temperature sensors (if available)

7) Systemd Integration

On Linux, the agent uses timer-based execution. The timer starts the associated oneshot service every 5 minutes.

Timer Unit (adminhelper-agent.timer)

[Unit]
Description=AdminHelper Agent Timer

[Timer]
OnBootSec=1min
OnUnitActiveSec=5min
RandomizedDelaySec=30
Persistent=true

[Install]
WantedBy=timers.target

Service Unit (adminhelper-agent.service)

[Unit]
Description=AdminHelper Agent
After=network-online.target
Wants=network-online.target

[Service]
Type=oneshot
ExecStart=/usr/local/bin/adminhelper-agent run --once
StandardOutput=journal
StandardError=journal
  • RandomizedDelaySec=30: Prevents simultaneous requests from many agents.
  • Persistent=true: Missed runs are caught up on the next boot.
  • Type=oneshot: The service runs once and exits.

8) Windows Integration

On Windows, the agent is registered as a native Windows service via sc.exe.

Property Value
Service Name AdminHelper-Agent
Binary C:\Program Files\AdminHelper\adminhelper-agent.exe run
Startup Type Automatic
Recovery Restart after 60 seconds (3 attempts) 
:: Register service manually (if not via 'service install'):
sc.exe create AdminHelper-Agent ^
  binPath= "C:\Program Files\AdminHelper\adminhelper-agent.exe run" ^
  start= auto ^
  DisplayName= "AdminHelper Agent"

:: Set recovery options:
sc.exe failure AdminHelper-Agent reset= 86400 ^
  actions= restart/60000/restart/60000/restart/60000

9) Troubleshooting

SSL Certificate Errors

For self-signed certificates, specify the CA certificate with --cacert /path/to/ca.crt. Alternatively, use --insecure for test environments.

Permission denied on /etc/frp

The agent requires root privileges for initial setup. Run commands with sudo.

frpc does not start

Check status with systemctl status frpc.service and journalctl -u frpc.service -n 50.

Monitoring push fails

Verify API_KEY and SERVER_ID in monitor.conf. Manual test: adminhelper-agent monitor push.

Diagnostic tip: Use adminhelper-agent run --once to manually trigger a single cycle and inspect the output directly in the terminal.
