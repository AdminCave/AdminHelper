Developer Guide

API Reference

All HTTP endpoints of the SRM server. Base URL: https://<host>/api. The complete OpenAPI schema lives at /api/docs (Swagger UI) and /api/openapi.json.

Authentication

Two mechanisms depending on the caller:

  • JWT bearer (web panel, desktop): Authorization: Bearer <access_token>. Refresh via httpOnly cookie.
  • API key (agents, extension, third parties): ?api_key=srm_… as query parameter or X-API-Key header.

Auth

POST/api/auth/login

Body: { "username": "admin", "password": "…" }. Response: { "access_token": "…", "user": {…} }. Sets the refresh_token cookie.

POST/api/auth/refresh

Reads the refresh cookie. Rotates the refresh token, blacklists the old jti. Reuse of a blacklisted token invalidates all user sessions.

POST/api/auth/logout

Blacklists the current refresh token, deletes the cookie.

GET/api/auth/me

Current user incl. roles, mfa_enabled.

Users

GET/api/users

List of all users (admin only).

POST/api/users

Create a user. Password is hashed via Argon2id.

PATCH/api/users/{id}

Change roles, email or is_active.

DELETE/api/users/{id}

Soft delete: is_active = false, sessions invalidated.

Connections

GET/api/connections

Query: ?kind=rdp|ssh|vnc|web, ?group=…, ?search=….

POST/api/connections

Body: { kind, name, host, port, tags, tunnel_id? }.

PUT/api/connections/{id}

Full replace.

DELETE/api/connections/{id}

Servers

GET/api/servers

Inventory, grouped by status.

POST/api/servers/{id}/token

24 h provision token for the agent. Redeemable once.

POST/api/servers/provision

Agent redeems the token → certificate + config.

API Keys

GET/api/apikeys
POST/api/apikeys

Body: { name, scope: [...], expires_at? }. Clear-text key returned once in the response; only SHA-256 is persisted.

DELETE/api/apikeys/{id}

Hooks

GET/api/hooks
POST/api/hooks

Body: { event, script, timeout_sec, enabled }.

GET/api/hooks/{id}/executions

Last 50 runs with exit_code, stdout, stderr.

POST/api/hooks/{id}/test

Run with a test event payload.

Monitoring

GET/api/monitoring/checks
POST/api/monitoring/checks

Body: { kind, server_id, config: {…}, threshold: {…}, interval_sec }.

POST/api/monitoring/ingest

Agent push. Body: Prometheus remote-write payload. Auth: X-API-Key with scope monitor_ingest.

GET/api/monitoring/query

SSRF-safe proxy to VictoriaMetrics. Only /api/v1/query and /api/v1/query_range are allowed.

Alerts

GET/api/alerts

Query: ?state=firing|acknowledged|resolved.

POST/api/alerts/{id}/ack

Acknowledge alert, triggers the alert.acknowledged hook.

FRP

GET/api/frp/tunnels
POST/api/frp/tunnels

Body: { name, type: "tcp"|"http"|"stcp", local_port, sk? }. stcp requires sk (shared key).

POST/api/frp/client-cert/rotate

Agent-initiated shortcut to /api/servers/{id}/cert.

Audit

GET/api/audit

Query: ?since=ISO, ?user_id=, ?action=. Paginate via ?limit + ?cursor.

Settings & Health

GET/api/settings

Global settings (branding, timeouts, feature flags).

GET/api/health

Returns { status, version, git_commit, db: "ok", redis: "ok" }. No auth.

Error Format

json
HTTP/1.1 422 Unprocessable Entity
Content-Type: application/json

{
  "detail": [
    { "loc": ["body", "port"], "msg": "ensure this value is less than 65536", "type": "value_error" }
  ]
}

Errors follow Pydantic validation conventions; 500 responses only contain { "detail": "internal error", "request_id": "…" } – the request ID correlates with the server log.

Rate Limits

  • Login: 5 failed attempts per 15 min / IP (Redis).
  • Refresh: 20 req/min / account.
  • Ingest: unbounded with a valid agent key, but samples are server-side deduplicated (by metric,labels,ts).
Use Swagger directly

/api/docs renders OpenAPI 3.1 interactively, including "Try it out". For CI clients, a single openapi-generator generate -g go -i https://srv/api/openapi.json is enough.
