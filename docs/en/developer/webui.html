Web Panel

The Web Panel is a standalone Svelte 5 SPA with a hash router. It lives under frontend-src/, is built via Vite, and is multi-stage-copied into the Python runtime image at Docker build time.

Project Structure

tree
frontend-src/
├── src/
│   ├── lib/
│   │   ├── api/           fetch wrapper for /api/*
│   │   ├── components/    Button, Modal, TagChip, Tabs, ...
│   │   ├── stores/        13 runes-based stores (auth, connections, ...)
│   │   ├── i18n/          DE/EN dictionaries
│   │   └── utils/
│   ├── routes/            Pages (hash-routed)
│   ├── App.svelte
│   └── main.ts
├── tests/e2e/             Playwright
├── svelte.config.ts
└── vite.config.ts

Routing

Hash router (no SSR, no SvelteKit client router): paths are #/connections, #/servers, etc. This works reliably behind any reverse proxy without SPA fallback support on the server side. Implementation in src/lib/router.ts.

Stores

Svelte 5 runes replace classic writables. Each store is a TypeScript class with $state fields and methods:

StorePurpose
authStoreLogin, logout, refresh, current user.
connectionsStoreLive list + search + group filter.
serversStoreServer inventory + provision token.
apiKeysStoreKey CRUD.
hooksStoreHook CRUD + execution log.
monitoringStoreChecks, alerts, VictoriaMetrics queries.
ansibleStorePlaybook runs, streaming output.
tunnelStoreFRP tunnel state and CRUD.
toastStoreGlobal toast notifications.
confirmStoreConfirmation dialogs.
languageStoreDE/EN switch (i18n).
themeStoreNo theme switch – always dark (from CSS tokens).
settingsStoreServer-side settings.

API Wrapper

All HTTP calls go through src/lib/api/client.ts. The client manages the access token and auto-triggers refresh on 401. A second 401 redirects to the login page.

uPlot Charts

Monitoring charts use uplot directly – no heavyweight chart framework. Wrapper in src/lib/components/charts/. uPlot is 30 KB gzipped and renders 100 k points without stutter.

Tests

Playwright smoke tests cover all 8 production pages: Login, Connections, Servers, Users, API-Keys, Hooks, Ansible, FRP Tunnels, Monitoring. Visual-diff screenshots are uploaded as CI artifacts.

bash
cd frontend-src
npm run check           # svelte-check + tsc
npm run lint            # eslint + svelte-eslint
npx playwright test     # E2E

Build & Deployment

The Vite build outputs to frontend-src/dist/. The Dockerfile copies this into /app/frontend/, and the FastAPI app mounts it as a static route. SPA fallback checks file existence before returning index.html – this prevents MIME errors on /assets/*.js.
