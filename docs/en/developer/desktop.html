Desktop Client

The Desktop Client is a Tauri 2 wrapper around a Svelte 5 SPA. The Rust backend (desktop/src-tauri/) exposes 24 typed commands; the frontend (desktop-src/) consumes them through a 1:1 bridge.

Architecture

tree
desktop/
├── src-tauri/
│   ├── src/
│   │   ├── main.rs           Tauri app setup
│   │   ├── commands/         24 #[tauri::command] (auth, connect, frp, ...)
│   │   ├── connection/       RDP/SSH/VNC/HTTP launcher
│   │   ├── frp/              FRP sidecar management
│   │   ├── keyring/          OS keyring (Linux: secret-service, macOS: keychain)
│   │   ├── http/             Reqwest client with auto-refresh
│   │   └── rdp/              Race guard with monotonic UUID
│   ├── tauri.conf.json
│   └── Cargo.toml
└── binaries/                 frpc sidecar per target triple
desktop-src/
├── src/
│   ├── lib/
│   │   ├── bridge/           invoke() wrapper per command
│   │   ├── stores/           10 stores (session, connections, ...)
│   │   ├── components/       Modals, widgets
│   │   └── pages/            4 pages (Dashboard, Connections, Monitoring, Ansible)
│   └── main.ts
└── tests/                    41 Vitest unit tests

Tauri Commands

The 24 commands cover auth, connection CRUD, connect flow, FRP tunnel, monitoring, Ansible, and settings. Each command is typed – the bridge uses TypeScript types that mirror the Rust structs exactly:

rust
#[tauri::command]
async fn connect_rdp(
    state: tauri::State<'_, AppState>,
    connection_id: String,
    password: String,
) -> Result<ConnectResponse, ConnectError> {
    // Race guard: monotonic UUID
    let connect_id = state.next_connect_id();
    ...
}

RDP Race Guard

Clicking Connect multiple times in quick succession would, without protection, spawn multiple RDP processes. The Desktop Client assigns a monotonic UUID per click. If the FRP tunnel setup is still running when a newer connect ID comes in, the older one is discarded. Implementation in desktop/src-tauri/src/rdp/race_guard.rs.

Keyring Secrets

Passwords are stored in the OS keyring, never in the SQLite DB:

  • Linux: secret-service (GNOME Keyring / KWallet)
  • macOS: Keychain
  • Windows: Credential Manager

Service prefix adminhelper:, account = <connection-id>.

FRP Sidecar

Tauri capabilities are scoped: frpc can only be spawned from the app binaries directory. At build time the matching target binary (frpc-x86_64-unknown-linux-gnu, frpc-aarch64-apple-darwin, …) must be dropped into desktop/src-tauri/binaries/.

Connect Flow

  1. User clicks Connect → connectFlow store sets state = 'connecting'
  2. Bridge calls connect_rdp (with race-guard UUID)
  3. Rust backend resolves the FRP tunnel (local visitor port)
  4. Password is fetched from the keyring or prompted via the PasswordPrompt modal
  5. RDP/SSH/VNC client is launched with a temporary config
  6. After exit the tunnel is torn down

Tests & Build

bash
cd desktop-src
npm run check                    # svelte-check
npm run test                     # vitest (41 tests)

cd ../desktop
cargo tauri dev                  # hot-reload dev mode
cargo tauri build --bundles deb  # DEB release
