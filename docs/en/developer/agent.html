Developer Guide

Go Agent

Unified Go agent under agent-go/. Replaces the legacy shell + Python agents and merges monitoring, FRP-client setup, and service installer into a single binary that runs on Linux (systemd) and Windows (service manager) alike.

Project Layout

tree
agent-go/
├── cmd/adminhelper-agent/   main.go – Cobra root
├── internal/
│   ├── cli/                 Subcommands (run, provision, monitor, ...)
│   ├── config/              agent.toml + env-var overrides
│   ├── http/                API client with auto-retry
│   ├── collectors/          cpu, ram, disk, smart, docker, zfs, proxmox
│   ├── frp/                 frpc management, cert rotation
│   └── service/             systemd / windowssvc integration
├── systemd/                 .service + .timer units
├── rpm/                     Spec file
├── build-deb.sh
├── build-rpm.sh
└── go.mod

Cobra Subcommands

CommandPurpose
versionEmit version + git commit (set via -ldflags).
provisionInitial setup: server URL, redeem token, fetch cert, enable services.
runOne full agent cycle (monitoring + FRP).
monitor runOnly the monitoring collector pass.
frpc rotateManually rotate the client certificate.
service install / uninstallsystemd-unit / Windows-service installation.

Collector Framework

Every collector implements a simple interface:

go
type Collector interface {
    Name() string
    Enabled(cfg *config.Config) bool
    Collect(ctx context.Context) ([]Sample, error)
}

type Sample struct {
    Metric string
    Value  float64
    Labels map[string]string
    Time   time.Time
}

Available collectors:

  • cpu – load + per-core utilization (from /proc/stat or GetSystemInfo on Windows)
  • ram – total/used/free/swap
  • disk – per mountpoint
  • smart – via smartctl --scan-open + --all --json=cv
  • docker – via Docker API socket
  • zfs – via zpool status -p
  • proxmox – via pveversion + API (on PVE hosts only)

systemd Timer with Jitter

Instead of cron, the agent uses systemd OnCalendar=*:0/5 + RandomizedDelaySec=30s. The jitter spreads load across the 5-minute window and avoids thundering-herd requests at the server.

Build

bash
cd agent-go
GOOS=linux   GOARCH=amd64 go build -o bin/adminhelper-agent     ./cmd/adminhelper-agent
GOOS=windows GOARCH=amd64 go build -o bin/adminhelper-agent.exe ./cmd/adminhelper-agent

VERSION=0.18.0 bash build-deb.sh
VERSION=0.18.0 bash build-rpm.sh
DEB/RPM Replaces

The DEB/RPM packages carry Replaces + Conflicts entries for the legacy packages srm-frpc-client, srm-monitor-agent and srm-agent. A single apt install adminhelper-agent migrates clean installs automatically.
